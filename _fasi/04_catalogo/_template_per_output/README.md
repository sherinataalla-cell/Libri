# `_template_per_output/` — Template di output per pubblicazione/marketing

Materiale derivato dalla scheda catalogo, finalizzato per **pubblicazione / marketing / presentazione editor**. Opzionale per il progetto: si compila solo se il progetto ne ha bisogno.

## File

| File | Cosa è |
|---|---|
| `TEMPLATE_descrizione_social.md` | descrizione narrativa-social per un'entità, in 7 registri d'uso (tag breve / scheda riga / paragrafo descrittivo / paragrafo evocativo / registri d'uso / cosa NON dire / frasi tipiche se parla) |

## Quando si compila

In Fase 04, **opzionalmente**, dopo che la scheda catalogo è in stato `canonico`. È un derivato della scheda, non un'alternativa.

Pattern: si compilano le descrizioni narrative-social a batch, dopo che il catalogo è completo o quasi, in vista di:

- creazione del sito web del progetto
- preparazione di materiale promozionale (newsletter, social, retro libro)
- presentazione editoriale a editor / agenti
- catalogo pubblicato online (es. atlante interattivo del mondo del progetto)

## Quando NON serve

Per progetti privati, prototipi non destinati a pubblicazione, o progetti dove le entità non vanno mai promosse separatamente: skip questa cartella.

## Pattern modulare

I 7 registri d'uso sono autosufficienti: ognuno può essere copiato e usato indipendentemente per il canale di destinazione. Non si è obbligati a compilarli tutti: il progetto sceglie quali servono.

Esempio: per un progetto destinato solo a un sito web statico, possono bastare A (tag breve), C (paragrafo descrittivo) e D (paragrafo evocativo). Gli altri sono opzionali.

## Vincoli

- **Mai inventare aggettivi non derivabili** dalla scheda catalogo
- **Mai citare spoiler narrativi** (la descrizione marketing non rivela archi narrativi futuri)
- **Mai usare il lessico AI bandito** del progetto
- **Mai contraddire la scheda catalogo** (ogni dato deve essere coerente con la fonte)
